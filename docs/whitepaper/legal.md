## 2. Risques Associés aux Tokens AKY

L'acquisition et la détention de tokens AKY comportent des risques significatifs. Vous devez comprendre et accepter les risques suivants avant toute interaction avec le protocole AKYRA.

### 2.1 Risques Technologiques

#### Bugs et Vulnérabilités de Smart Contracts

Malgré plusieurs audits (PeckShield, Code4rena, Quantstamp), **aucun code n'est exempt de bugs**. Des vulnérabilités peuvent être découvertes après le déploiement, entraînant :
- Perte totale des tokens déposés
- Exploitation par des acteurs malveillants
- Nécessité de migration vers de nouveaux contrats

**Aucun recours légal n'est garanti en cas de perte due à un bug.**

#### Risque de Consensus Blockchain

AKYRA Chain est un L2 Optimistic Rollup sur Ethereum. Les risques incluent :
- Défaillance du séquenceur (centralisé en Phase 1)
- Attaque 51% sur Ethereum L1 (improbable mais possible)
- Challenge period fraud (bien que mitigé par le modèle optimistic)

#### Dépendance à l'Infrastructure Externe

AKYRA dépend de :
- Ethereum Mainnet (settlement layer)
- Celestia (data availability)
- Providers RPC (Conduit/Caldera en mainnet)
- LLM providers (OpenAI, Anthropic, etc.)

**La défaillance de l'un de ces éléments peut rendre le protocole temporairement ou définitivement inutilisable.**

### 2.2 Risques de Marché

#### Volatilité Extrême

Les crypto-actifs sont **extrêmement volatils**. Le prix du AKY peut fluctuer de +/- 50% ou plus en une seule journée.

**Exemples historiques** :
- Luna/UST : -99.9% en 48h (Mai 2022)
- FTT (FTX) : -95% en 72h (Nov 2022)
- VIRTUAL (Virtuals Protocol) : -82% depuis ATH (Jan 2025)

**Vous pourriez perdre 100% de votre investissement.**

#### Liquidité Limitée

Surtout en phase de lancement, la liquidité sur les DEX et CEX peut être **insuffisante** pour vendre de grandes quantités sans impact de prix massif (slippage >20%).

#### Manipulation de Marché

Les marchés crypto sont sujets à :
- Pump & dump
- Wash trading
- Spoofing
- Manipulation par baleines

**AKYRA ne peut contrôler le comportement des traders.**

### 2.3 Risques Réglementaires

#### Incertitude Juridique

La régulation crypto évolue rapidement et varie par juridiction :
- **Europe** : MiCA (Décembre 2024)
- **USA** : SEC vs Ripple, Coinbase, Binance
- **Asie** : Interdictions en Chine, régulation stricte en Inde

**AKYRA pourrait être considéré comme illégal dans certaines juridictions sans préavis.**

#### Classification en Security Token

Si un régulateur (SEC, AMF, BaFin) classe AKY comme **security**, cela entraînerait :
- Obligation de licence (coûteuse et complexe)
- Delisting des exchanges non régulés
- Amendes et poursuites potentielles
- Effondrement du prix du token

**Bien que nous ayons conçu AKY pour éviter cette classification, aucune garantie n'existe.**

#### KYC/AML

Les régulations KYC/AML (Know Your Customer / Anti-Money Laundering) pourraient forcer AKYRA à :
- Identifier tous les utilisateurs
- Bloquer certains utilisateurs (sanctions OFAC)
- Rapporter les transactions suspectes

**Cela pourrait compromettre l'anonymat et la décentralisation.**

#### Échec de Développement

Le projet AKYRA pourrait échouer à :

- Générer assez de revenus pour sustainer l'infrastructure

**Dans ce cas, le projet serait abandonné et le token AKY perdrait toute valeur.**

#### Départ de l'Équipe

Les membres clés de l'équipe pourraient :
- Quitter le projet
- Être incapacités (maladie, décès)
- Être poursuivis légalement

**Bien que le code soit open source, la perte de l'équipe réduirait drastiquement les chances de succès.**

#### Concurrence

D'autres projets similaires (Virtuals, Autonolas, Fetch.ai, nouveaux entrants) pourraient :
- Capturer plus de marché

- Avoir plus de capital

**AKYRA pourrait devenir obsolète.**
